## 1. 产品概述

| 角色 | 占比 | 核心需求 |
|------|------|---------|

## 3. 功能需求

### 3.1 用户系统
- 微信授权一键登录，后端调 jscode2session 换取 openid
- 返回 JWT Token 用于后续接口鉴权
- 首次登录自动创建用户账号

### 3.2 实名认证
- 填写学号、姓名、上传学生证照片
- 提交后由管理员后台审核
- 认证通过后获得发布/接单权限

### 3.3 订单系统
- **发布订单**：选择跑腿类型（代取快递/代买物品/代送物品/其他）、填写地址、设置金额（2~50元）、可选图片
- **订单大厅**：浏览所有待接单的订单
- **抢单**：使用Redis分布式锁防止多人同时接同一单
- **状态流转**：待支付 → 待接单 → 已接单 → 进行中 → 待确认 → 已完成

### 3.4 智能匹配
- 跑腿员上线时 GEOADD 记录位置
- 发布订单时 GEORADIUS 搜索 3km 内跑腿员
- 按距离权重0.5 + 信用分权重0.3 + 接单率0.2 排序推荐

### 3.5 支付
- 支付宝沙箱集成
- 订单支付后进大厅等待接单
- 异步回调验签 + 幂等处理

### 3.6 钱包
- 跑腿员接单后佣金冻结
- 订单完成自动结算到可用余额
- 提现申请（最低10元，日限额500元）

### 3.7 聊天
- 订单绑定，仅订单双方可聊天
- 支持文字消息，消息已读状态

### 3.8 评价
- 订单完成后双方互评
- 评分1~5分，信用分联动：高分+1，低分-2
- SQL 原子更新防止并发竞态

### 3.9 管理后台
- 数据大屏：用户数、订单数、今日订单、在线用户、订单趋势图
- 用户管理：列表、搜索、封禁/解封、删除
- 订单管理：订单列表、状态筛选
- 实名审核：通过/拒绝
- 申诉仲裁：处理纠纷订单

## 4. 技术实现

| 模块 | 实现方案 |
|------|---------|
| 后端框架 | Spring Boot 3.2.5 + MyBatis-Plus + Lombok |
| 安全 | JWT + BCrypt + CORS白名单 |
| 数据库 | MySQL 8（业务）+ Redis 6（GEO/锁/缓存） |
| 前端 | UniApp（小程序）+ Vue 3 + Element Plus（后台） |
| 支付 | 支付宝沙箱 SDK |
| 匹配 | Redis GEOADD + GEORADIUS |
| 锁 | Redis setIfAbsent + Lua 脚本 |

### 4.1 抢单锁设计
```
setIfAbsent(lockKey, runnerId, 30s) → 获取锁
   ↓ 成功
校验订单状态 → 执行业务
   ↓ finally
Lua脚本: if get(key)==runnerId then del(key) end
```

### 4.2 信用分原子更新
```sql
UPDATE users SET credit_score = GREATEST(0, LEAST(100, credit_score + delta)) WHERE id = ?
```

### 4.3 支付宝回调
```
验签 → 幂等检查 → 金额校验 → 更新订单状态 → 返回success
```

## 7. 开发难点

| 难点 | 解决方案 |
|------|---------|
| 多人抢单超卖 | Redis分布式锁 + Lua脚本原子释放 |
| 支付宝重复回调 | 验签 + 幂等处理 |
| 信用分并发竞态 | SQL原地加减替代 read-calc-write |
| 配置泄露 | 环境变量代替明文 |
| 订单越权操作 | JWT解析userId比对订单归属 |
| 评价未完成订单 | 提交评价前校验订单状态是否为已完成 |
