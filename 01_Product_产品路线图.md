# 校园互助跑腿小程序 - 产品路线图

## 📋 产品概述

| 项目信息 | |
|---------|-----|
| **产品名称** | 校园帮 - 互助跑腿小程序 |
| **产品定位** | 面向高校校园的C2C互助跑腿服务平台 |
| **目标用户** | 在校大学生、教职工 |
| **核心价值** | 高效连接供需双方，解决校园"最后一公里"跑腿需求 |
| **商业模式** | 平台抽取订单服务费（默认5%） |

---

## 🎯 战略规划

### 愿景
成为国内领先的校园生活服务平台，连接千万师生，让校园生活更便捷。

### 使命
通过技术手段优化校园资源配置，让每一位同学的时间都更有价值。

### OKR

**Q2 2026 目标：完成MVP开发并上线**
- 关键结果1：完成后端核心功能开发（用户、订单、支付、匹配）
- 关键结果2：完成微信小程序前端开发并通过审核
- 关键结果3：完成管理后台开发并可运营
- 关键结果4：完成支付、地图等第三方服务集成

**Q3 2026 目标：用户增长与产品迭代**
- 关键结果1：累计注册用户达到目标院校的50%
- 关键结果2：日活跃订单量达到100+单
- 关键结果3：用户留存率达到30%以上
- 关键结果4：完成3个版本迭代优化

## 📑 依赖项清单

**必须配置的第三方服务：**
- [ ] 微信小程序账号（获取AppID、AppSecret）
- [ ] 支付宝开放平台账号（沙箱环境用于测试）
- [ ] 腾讯地图/高德地图API（用于地理位置服务）
- [ ] 百度AI OCR接口（用于学生证识别）
- [ ] 阿里云/腾讯云服务器（用于部署）

**技术环境要求：**
- [ ] JDK 17+
- [ ] MySQL 8.0+
- [ ] Redis 6.0+（必须支持GEO命令）
- [ ] Node.js 16+
- [ ] Maven 3.6+
- [ ] 微信开发者工具
