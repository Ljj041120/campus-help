### 任务 3：修复分布式锁释放风险

**目标**：使用 Lua 脚本保证只能释放自己持有的锁，防止误删他人锁。

**修改文件**：`backend/src/main/java/com/campushelp/service/OrderService.java`

**当前代码**（`OrderService.java:161`）：
```java
finally {
    redisTemplate.delete(lockKey);  // 无条件删除，不校验持有者
}
```

**改动说明**：

```java
// 修改后（Lua 脚本原子操作：只有锁的 value 匹配时才删除）
finally {
    String luaScript = 
        "if redis.call('get', KEYS[1]) == ARGV[1] then " +
        "  return redis.call('del', KEYS[1]) " +
        "else " +
        "  return 0 " +
        "end";
    redisTemplate.execute(
        new DefaultRedisScript<>(luaScript, Long.class),
        Collections.singletonList(lockKey),
        runnerId.toString()
    );
}
```

> **额外建议**：考虑使用 Redisson 的 `RLock`，它内置看门狗自动续期 + 正确的锁释放语义，避免自己处理 Lua 脚本。

@Scheduled(fixedDelay = 60000)
public void cancelTimeoutOrders() {
    String lockKey = "job:order_timeout:lock";
    String lockValue = instanceId;  // 实例唯一标识
    Boolean acquired = redisTemplate.opsForValue()
        .setIfAbsent(lockKey, lockValue, 55, TimeUnit.SECONDS);  // TTL < fixedDelay
    if (Boolean.TRUE.equals(acquired)) {
        try {
            // ... 原有取消逻辑 ...

    }
}

### 任务 14：完善评价系统（信用分增减 + 并发安全）

**目标**：补充低分扣分逻辑，并用数据库原子操作消除竞态条件。

// 修改后：根据评分，用一条原子 SQL 完成信用分调整
public void submitEvaluation(Long orderId, Long fromUserId, Long toUserId, int score, String content) {
    // ... 保存评价记录 ...

    if (toUserId == null) return;

    int delta;
    if (score >= 4) {
        delta = 1;                            // 高分 +1
    } else if (score <= 2) {
        delta = -2;                           // 低分 -2
    } else {
        delta = 0;                            // 3分不变

    }
}

### 任务 19：Admin 端封装统一 HTTP 客户端

**目标**：消除裸 `fetch` 散落各处、`token()` 重复定义 7 次的问题。

**修改文件**：新建 `admin/src/api/request.js`，修改全部 `views/*.vue`

**新建 `admin/src/api/request.js`**：

```javascript
import { ElMessage } from 'element-plus'

const BASE_URL = '/api'

const request = async (url, options = {}) => {
  const token = localStorage.getItem('adminToken')
  const res = await fetch(BASE_URL + url, {
    ...options,
    headers: {
      'Authorization': token ? `Bearer ${token}` : '',
      ...(options.headers || {})
    }
  })

  if (res.status === 401) {
    localStorage.removeItem('adminToken')
    location.href = '/login'
    throw new Error('未登录')
  }

  const data = await res.json()
  if (data.code !== 200) {
    throw new Error(data.message || '请求失败')
  }
  return data.data
}

export default request
```
